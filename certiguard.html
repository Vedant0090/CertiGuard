How CertiGuard works
⛓️
Tier 1 — Hash Verification
Every certificate gets a unique SHA-256 fingerprint (hash). When a university issues a certificate, its hash is stored in the database. To verify, the same file is hashed again and compared — even one pixel change produces a completely different hash, instantly detecting tampering.
🔬
Tier 2 — AI Forensic Analysis
For legacy certificates without a hash record, AI analysis runs automatically. Tesseract OCR extracts text to find certificate keywords and institution names. OpenCV detects circular seals, signature regions, and analyses noise patterns for copy-paste tampering artifacts.
What each AI check does
OCR Extraction
Tesseract reads all text from the image — finds names, roll numbers, dates, institution names.
Keyword Check
Scans for 30+ certificate keywords like "certify", "awarded", "degree", "examination".
Seal Detection
OpenCV's Hough Circle Transform finds circular stamps and seals in the image.
Signature Check
Finds dark flowing strokes in the lower half of the image typical of signatures.
Tampering Check
Analyses noise consistency across image grid — inconsistent patterns indicate copy-paste edits.
Institution Match
Checks text against a database of known Indian universities, IITs, NITs, and boards.
Tech stack: Python · Flask · Tesseract OCR · OpenCV · NumPy · PyPDF2 · Pillow · SHA-256 hashing
